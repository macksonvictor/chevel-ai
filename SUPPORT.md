# Support

Use GitHub Issues for public bugs, feature requests, and documentation improvements that do not include private information.

## Public Issues

Good public issue topics:

- chat UI bugs;
- local setup problems;
- API behavior;
- tests or CI failures;
- documentation gaps;
- safe Dum-E/U bridge improvements.

Do not include:

- access tokens;
- private hardware addresses;
- private model files;
- personal data;
- unreleased robot credentials;
- security vulnerabilities.

## Security

For security-sensitive reports, use [SECURITY.md](./SECURITY.md).

## Before Opening An Issue

Please include:

- operating system;
- Python version;
- command used to start CHEVEL;
- relevant endpoint or UI flow;
- expected behavior;
- actual behavior;
- minimal logs without secrets.
